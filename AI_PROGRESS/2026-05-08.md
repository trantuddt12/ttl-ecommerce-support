### Summary

Full payment module implementation following `docs/payment-roadmap/` (files 00‚Äì11). Covers all 5 phases:
1. **Domain foundation** ‚Äî entities, state machine, invariants, REST API
2. **Reliability** ‚Äî idempotency with conflict detection + outbox transactional publish
3. **Distributed workflow** ‚Äî orchestrated checkout saga with compensation and event dedup
4. **Finance operations** ‚Äî EOD reconciliation with mismatch classification + chargeback lifecycle
5. **Observability** ‚Äî outbox lag metric, publish fail counter, saga stuck detection via scheduled queries

Provider gateway (VNPay/Momo/Stripe) is stub/mock. Next step is implementing `ProviderGatewayService` with Resilience4j (circuit breaker + retry + timeout) per file `06-2pc-poc-production-hardening.md`.
